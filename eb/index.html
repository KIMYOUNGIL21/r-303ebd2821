상위 계정 공통 승리 공식 6가지

1. 팔로워 엔진 = 무료 전자책 배포 이벤트

글 잘 써서 크는 게 아니라, 배포 이벤트로 점프한다. 공식: 「좋아요 + 팔로우 + 리포스트 + 댓글에 키워드 → DM으로 전달」. 리포스트 조건이 핵심 — 받은 사람의 팔로워에게 글이 퍼진다.

시애나: "내가 쓴 9.9만원 전자책 그냥 푼다. 크몽에서 수천만원 팔린 자료인데… 팔로우+댓글 '신청'" / "7일 뒤에 유료화 예정이야 서둘러"

2. 프로필 바이오 = 숫자 증거 한 줄

바이오에 실적 숫자가 없으면 아무도 안 믿는다. 크기보다 구체성이 중요하다.

"AI음악 3만곡 제작, 200곡 발매 중" (지니비니) / "전자책 펀딩 3회 성공, 최고 420%" (예글)

3. 글 구조 = 도발 첫 줄 + 번호 리스트 + 행동 지시

첫 줄은 확신체·도발로 스크롤을 멈추고, 본문은 번호 리스트로 읽기 쉽게, 마지막은 반드시 행동 지시("저장해", "오늘 하나 만들어봐")로 닫는다. 존댓말 설명체는 상위 계정에 없다.

"어차피 알려줘도 100명 중 90은 안 하더라" (지니비니) / "아직도 스레드로 돈 못 번다고?" (시애나)

4. 과정 공개(빌드 인 퍼블릭)가 판매 페이지보다 잘 판다

책 만드는 과정, 채널 성장 숫자, 실패 경험을 실시간으로 공개하면 구경꾼이 예비 구매자가 된다.

온도: 전자책 목차를 투표로 정함 / 예글: 펀딩 진행 상황 중계 / 퀀텀점프: 운영 사례와 ROI만 공개

5. 프로필 링크 = 링크 허브 (단일 링크 ❌)

상위 계정은 litt.ly(리틀리) 같은 허브에 무료책·유료책·오픈채팅·채널을 한 페이지로 모은다. 방문자가 어떤 단계든 다음 행동이 있게.

시애나 → litt.ly/sienna__ai / 예글 → litt.ly/yegle

6. 커뮤니티 장치 (챌린지·오픈카톡)

팔로워가 어느 정도 모이면 챌린지나 단톡방으로 묶는다. 참여자의 인증글이 무료 광고가 되는 구조.
